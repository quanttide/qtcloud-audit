# 审计规则沙盒

1. 规则沙盒（Rule Sandbox）
允许 AI 在隔离环境中运行新规则，对比旧规则的发现数量 vs 误报率。
向用户展示：“若采用此规则，上月会多产生 200 条告警，其中 180 条是您曾标记为‘正常’的场景。”
作用：用数据说服人，也让人理解 AI 的权衡。
2. 质疑日志（Challenge Log）
记录每一次人的质疑：
质疑时间、用户身份
被质疑的规则版本
质疑类型与理由
AI 后续的调整动作
作用：形成组织级的“审计规则进化史”，供元审计委员会复盘。
3. 噪音仪表盘（Noise Dashboard）
核心指标：
误报率（False Positive Rate）
平均处理时间（MTTR for False Alarms）
用户抑制率（% of alerts manually suppressed）
当噪音指标超标时，自动提醒：“当前规则集可能过于敏感，建议启动校准会议。”
作用：将主观感受（“太多告警了！”）转化为客观指标，驱动制度迭代。
